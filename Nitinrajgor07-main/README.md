## About Me

I'm a Computer Science graduate student and software developer working at the point where **applied machine learning meets production-grade backend engineering**. My starting point is usually a research question — how do you price a toll dynamically based on traffic conditions, how does an agent learn to win at Ludo — and my finishing point is a working, deployable system with a real interface around it.

**Professional story.** I completed my BCA at Nav Gujarat College, Gandhinagar (Gujarat University), and I'm now in the third semester of an MSc in Computer Science & IT at Jain (Deemed-to-be) University, Bengaluru. Alongside my degree, I'm interning at Innovexis, and I split my project work between two threads that reinforce each other: **Java backend engineering** (Spring Boot, JDBC, OOP-driven system design) and **applied AI/data work** (scikit-learn, OpenCV, reinforcement learning). Two of my projects have been formalized into published research papers rather than staying as one-off builds.

**Career goals.** I'm looking for a fresher Java Developer or Software Development role where I can work on backend systems at production scale, ideally somewhere that also values the applied-ML side of my background rather than treating it as a separate track.

**Currently learning.** Deepening my Spring Boot and Spring Security fundamentals, sharpening SQL and system design basics, and continuing to formalize applied-ML side projects into complete, documented systems.

### 🚦 Predictive Smart Toll Pricing System

<div align="center">
  <img src="./illustrations/toll_pricing_architecture.png" width="90%" alt="Predictive Smart Toll Pricing System Architecture" />
</div>

<br />

A dynamic, congestion-aware toll-pricing engine built directly from my published research on predictive smart toll pricing. A Random Forest model (scikit-learn) estimates real-time congestion-adjusted pricing, served through a Flask backend with a dark-themed dashboard.

#### 🏗️ System Architecture
The application follows a modular system architecture separating predictive modeling, API communication, and the visualization UI:
1. **Predictive Engine**: A Random Forest Regressor trained on traffic density indices, peak hours, and weather coefficients.
2. **REST API**: Flask backend providing predictions via RESTful endpoints (`/api/predict-toll`).
3. **Analytics Dashboard**: A dark-theme interactive dashboard rendering live price fluctuations and historic trends using Chart.js.

#### 📦 Project Components
* `app.py` - Flask web server and API endpoints.
* `model.py` - Machine learning training, evaluation, and serialization scripts.
* `templates/dashboard.html` - Dark-mode dashboard visualizing toll patterns.

#### ⚙️ Prerequisites
* Python 3.9+
* Libraries: `flask`, `scikit-learn`, `pandas`, `numpy`, `joblib`
* Browser: Modern web browser with Javascript enabled.

### 📈 Indian Stock Market Paper Trading Dashboard

<div align="center">
  <img src="./illustrations/stock_market_architecture.png" width="90%" alt="Stock Market Paper Trading Dashboard Architecture" />
</div>

<br />

A Streamlit-based paper trading platform for the Indian stock market, featuring sector-wise analytics tabs, a virtual balance system, and AI-generated market commentary powered by the Claude API.

#### 🏗️ System Architecture
1. **Interactive Client**: A Streamlit UI layout displaying sector trends, current portfolios, and transaction logs.
2. **AI Commentary Engine**: Integration with Anthropic's Claude API to parse stock performance data and generate structured market insights.
3. **Ledger System**: In-memory portfolio tracking simulating buy/sell orders and updating virtual balances.

#### 📦 Project Components
* `main.py` - Streamlit app configuration, navigation tabs, and layout design.
* `portfolio.py` - In-memory database logic managing buy/sell transactions and virtual wallets.
* `ai_commentary.py` - Connection client for Claude API prompt engineering.

#### ⚙️ Prerequisites
* Python 3.9+
* Libraries: `streamlit`, `anthropic`, `pandas`, `plotly`
* Credentials: An active `ANTHROPIC_API_KEY` configured in environment variables.

### 🧑‍💻 Face Recognition Attendance System

A desktop attendance system using the LBPH face-recognition algorithm, with a Tkinter interface and a SQLite backend for attendance records.

#### 🏗️ System Architecture
1. **Haar Cascade Face Detector**: Captures live frames via OpenCV, detecting faces in the web camera feed.
2. **LBPH Recognition Engine**: A Local Binary Patterns Histograms classifier that pre-processes student face data, trains local models, and recognizes identities in real-time.
3. **Database Logger**: SQLite database registering attendance logs and timestamps on successful match matches.

#### 📦 Project Components
* `attendance.py` - Main Tkinter application running the live camera feed and recognizer.
* `dataset_generator.py` - Captures and registers sample face photos for training.
* `train_model.py` - Compiles samples into the LBPH model trainer.

#### ⚙️ Prerequisites
* Python 3.8+
* Libraries: `opencv-python`, `opencv-contrib-python`, `numpy`, `pillow`
* Hardware: Webcam (built-in or USB external)
